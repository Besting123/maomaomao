# 喵伴云养 —— 开发计划文档

> 本文档基于最新项目目标（goal.md）制定，用于指导后续 UI 概念设计开发。

## 开发阶段规划

### 第一阶段：理念转型与基础改造

**目标：** 将产品从"投喂中心"转型为"陪伴中心"，完成核心框架调整。

#### 1.1 导航与定位调整

| 当前 Tab | 调整后 Tab | 说明 |
|----------|-----------|------|
| 首页 | 首页 | 保留，增加签到/任务入口 |
| 校园 | 校园 | 保留，增强教育引导内容 |
| **投喂** | **陪伴** | 重命名并重新定位整个页面 |
| 论坛 | 论坛 | 基本保留 |
| 我的 | 我的 | 扩展成就/学习维度 |

#### 1.2 首页改造
- 保留现有的 Hero 区域、猫咪动态卡片、故事集
- 新增"每日签到"入口卡片
- 新增"今日学习任务"提示卡片
- 将"核心功能入口"中的"投喂猫咪"改为"云陪伴"
- 新增"我关注的猫咪"快报区域

#### 1.3 FeedScreen → CompanionScreen 改造
- 页面标题从"投喂"改为"陪伴"
- 保留猫咪状态展示卡片
- 将"补水"/"喂食"按钮扩展为多种互动方式（安抚、观察、记录、补水、喂食）
- 保留线下联动入口
- 新增"陪伴记录"区域

---

### 第二阶段：新功能模块开发

**目标：** 实现计划中完全缺失的核心功能。

#### 2.1 科学互动与新手教育页面（新增页面/组件）
- 知识卡片浏览（图文形式）
  - 如何判断猫咪是否愿意被接近
  - 如何识别猫咪行为和情绪
  - 正确投喂与补水指南
  - 逗猫与安抚技巧
  - 危险行为警示
- 答题/测验组件
- 学习进度追踪

#### 2.2 签到与任务系统
- 每日签到 UI（连续签到激励展示）
- 任务列表页面（学习任务、观察任务、陪伴任务）
- 任务完成状态与奖励展示

#### 2.3 代币经济系统 UI
- 代币余额展示组件（首页 + 个人中心）
- 代币获取记录
- 代币兑换商店 UI（兑换虚拟互动行为）

---

### 第三阶段：现有页面增强

**目标：** 在已有的高完成度页面上补充新计划要求的细节。

#### 3.1 校园地图增强
- 区域详情中增加"接近指南"内容
- 不同时段的互动建议差异化
- 增加"不应打扰"场景提示

#### 3.2 猫咪档案增强
- 新增"关注此猫"按钮
- 新增"我的陪伴记录"区域
- 增加情绪状态指示器
- 增加"互动边界"专区（不同猫咪的接近方式不同）
- 增加成长记录展示

#### 3.3 个人中心增强
- 善意账本扩展：增加签到天数、学习完成数、任务参与数
- 新增代币余额展示
- 新增学习记录/知识勋章
- 陪伴轨迹扩展：涵盖学习、签到、互动等类型
- 修正学校名称为"北京交通大学"

#### 3.4 论坛微调
- 增加"组队活动"帖子类型
- 增加"知识分享"内容分类

---

### 第四阶段：体验打磨

**目标：** 整体体验一致性与细节优化。

- 统一"陪伴"叙事语言，替换所有以"投喂"为中心的文案
- 新增互动微动画（签到成功、任务完成、陪伴反馈等）
- 猫咪虚拟形象交互动效
- 整体 UI 一致性审查与优化

## 优先级排序

| 优先级 | 任务 | 理由 |
|--------|------|------|
| P0 | 导航重命名 + FeedScreen 转型 | 核心理念落地的第一步 |
| P0 | 首页增加签到/任务入口 | 新用户首次体验的关键触点 |

| P2 | 校园地图教育增强 | 提升地图的教育价值 |
| P2 | 个人中心扩展 | 成就感与回顾的完善 |

| P3 | 组队机制 | 社交功能可后续迭代 |

## 技术栈

- **平台：** Android (Kotlin)
- **UI 框架：** Jetpack Compose + Material 3
- **导航：** Jetpack Navigation Compose
- **状态管理：** Compose State + ViewModel（已引入，StateFlow 驱动全局状态）
- **数据：** 当前为内存模拟数据（ViewModel 层），后续可对接 BJTU 猫谱小程序数据
